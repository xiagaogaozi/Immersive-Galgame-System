# 沉浸式galgame系统功能总集表

本文件是项目根目录的工程总览表，用于让人类和 AI 在实现前先确认功能边界、模块归属、UI 入口、数据来源和验收方向。

| 生图插件 | 生成队列与轮询设置 | 用户配置 | 否 | 否 | 否 | 超时、轮询间隔、失败反馈、当前楼层图片池。 |
| 可选组件 | 组件启用与排序 | 用户配置 / Mod | 是 | 是 | 是 | 组件 Mod 在本页或对应能力页管理，不提供总 Mod 管理页。 |
| 选项浮窗 | 选项组件 | Mod | 是 | 是 | 是 | 点击后走输入框发送。 |
| 选项浮窗 | 选项解析规则 | Preset | 是 | 是 | 是 | 可解析模型输出中的选项。 |
| shujuku 表格 | 表格浏览/编辑 | 内置功能 | 否 | 否 | 否 | 不重复 shujuku 模板导入。 |
| UI 预设 | 系统 UI/CSS | Preset | 是 | 是 | 是 | 覆盖阅读器、工具栏、对话气泡、名字牌、头像、选项、设置页、shujuku 表格页。 |
| UI 预设 | UI Skin / 布局预设 | Preset | 是 | 是 | 是 | 支持全屏字幕式、左上工具栏、右上工具栏、底部对话栏等完全不同 UI。 |
| 快捷键 | 快捷键绑定 | 用户配置 | 否 | 否 | 否 | 只手动编辑，不做预设包。 |

## 5. 能力分组导入模型

所有能力页使用相同交互模型：

```text
[当前启用项下拉框] [编辑] [导入] [导出] [设置]

默认预设
├── 内置项
用户导入
├── 用户项
本地草稿
└── 未导出项
```

不存在全局 Mod 管理页。导入合集时由 `import-router` 按 `type` 拆分到对应能力页：

| `type` | 归属页面 |

| `image-request-builder-preset` | 生图插件页 |
| `workflow-preset` | 生图插件页 |
| `ui-component` | 可选组件页 |
| `choice-component` | 选项浮窗页 |
| `theme-preset` / `css-preset` | UI 预设页 |
| `ui-skin-preset` / `ui-layout-preset` | UI 预设页 |
| `scene-regex-preset` | 正则与正文页 |
| `text-filter-preset` | 正则与正文页 |
| `text-format-preset` | 正则与正文页 |

## 6. 优先级路线

| 阶段 | 目标 | 说明 |
| --- | --- | --- |
| P0 | app 工程 + 远程 bundle 骨架 | 先跑通主工程构建与 GitHub 发布形态；loader 后续按真实需要补。 |
| P1 | 迁移 Visual Novel 既有阅读器 | 保留原有使用体验。 |
| P2 | PresetRegistry + 能力页下拉管理器 | 先把标签解析、正文格式化、时空栏正则预设化。 |
| P3 | Host / Action / Component / Storage 底座 | 先把输入框发送、动作、组件、IndexedDB 和导入分发稳定下来。 |
| P4 | 背景/立绘分层 | 建立背景、立绘、头像、环境效果、规则匹配。 |
| P5 | 生图插件页 + ImageProvider API | 把现有 chatu8/chami/NAI 统一成 provider，补 prompt adapter、request builder 和轮询队列。 |
| P6 | shujuku 表格页 | 做内置可编辑前端。 |
| P7 | 选项浮窗与输入框发送 | 保障 shujuku 剧情推进触发。 |
| P8 | UI/CSS 全量预设 | 所有页面统一样式覆盖。 |
| P9 | 模拟测试与性能预算 | 补齐 AI 友好防回归工作流。 |
