# v0.2.12 原版 VN 可用性修复施工图

## Summary

- 本轮目标不是新增功能，而是修复当前导入后“入口能打开但阅读器不可用”的问题。
- 最高优先级是恢复原版 Visual Novel 的可用链路：魔法棒入口视觉一致、打开后读取正确楼层正文、阅读器显示真实正文而不是酒馆 UI HTML、输入发送可用。
- 本轮继续不修改 `projects/Visual Novel/**` 原版项目；只在 `projects/Visual Novel/` 内对照迁移。
- 本轮实施后应发布运行时新版本 `v0.2.12`；本施工图文件本身只是计划归档，不代表当前 runtime 已修复。

## 截图诊断

截图可见现象：

- 阅读器黑底已打开，说明 loader 和魔法棒点击链路大概率已执行到 `openLatestAvailable()`。
- 底部 `.vnm-dialog` 显示了 `<div class="flex-container">`、`API Connections`、`rightNavHolder` 等宿主 UI HTML，说明上游正文抽取把酒馆页面 UI 或 HTML 字符串当成了楼层正文。
- 左侧可见类似加载/装饰的小图标，但没有出现可读 VN 正文；这是文本 payload 错误，不是单纯 CSS 问题。
- 魔法棒入口图标与原版不一致：当前是 `fa-wand-magic-sparkles`，原版是 `fa-book-open`。

## UI 预期

1. 魔法棒菜单中只能出现一个入口，视觉与原版一致：
   - DOM：`a.list-group-item vnm-magic-entry`
   - id：`vnm-magic-entry-btn`
   - data：`data-vnm-magic-entry="1"`、`data-vnm-version="<IGS版本>"`
   - icon：`<span class="fa-solid fa-book-open" aria-hidden="true"></span>`
   - visible text：`Visual Novel`
   - title/aria-label：`打开 Visual Novel`
2. 点击入口后，阅读器必须显示当前最新可读楼层的正文，不得显示酒馆设置按钮、抽屉按钮、导航 HTML 或 `<div ...>` 字符串。
3. 如果最新消息没有 `<content>` 标签，但可见楼层正文存在，应按原版 `visibleText` / `cleanNarrativeSource` fallback 得到正文。
4. 如果消息有 `<content>` 标签，则优先读取 `<content>` 内正文，并排除 `thinking/Subtext_think/Status_block/text_to_image/parallel_world/aftertalk`。
5. 如果消息使用 `@bubble:` 格式，默认按原版规则格式化为 `[角色]：正文`。
6. 阅读器仍保留 `#vnm-overlay`、`.vnm-dialog`、`.vnm-ctrl-bar`、`.vnm-controls`、`#vnm-input`、`#vnm-send-btn` 等原版 DOM。
7. 输入框 `Enter` 发送、`Shift+Enter` 不发送；发送后不关闭阅读器。

## 不等价即失败

- 入口继续显示 `fa-wand-magic-sparkles`。
- 入口 visible text 继续显示 `Visual Novel`。
- 酒馆助手脚本按钮区出现额外按钮。

- `openLatestAvailable()` 能返回 `{ ok:true }`，但 `.vnm-text` 为空或只显示 HTML。
- 只在测试里替换字符串，而没有改正文抽取/清洗链路。

## 施工步骤

   - 默认 label 改为 `Visual Novel`。

   - 新建按钮时使用 `id="vnm-magic-entry-btn"`、`class="list-group-item vnm-magic-entry"`、`data-vnm-magic-entry="1"`、`data-vnm-version="<version>"`。
   - `innerHTML` 固定为 `<span class="fa-solid fa-book-open" aria-hidden="true"></span> Visual Novel`。
   - 为兼容已有清理逻辑，可同时识别并删除旧 `[data-igs-magic-entry]`，但新入口不要再生成 `igs-magic-entry`。
2. 修改 `app/src/core/bootstrap.js`：
   - 删除对魔法棒入口传入 `label: 'Visual Novel'`，或改为 `label: 'Visual Novel'`。
3. 修改 `app/src/host/CONTRACT.md`：
   - 将“注入 `Visual Novel` 菜单项”改为“注入原版 `Visual Novel` 菜单项”。
4. 修改测试：
   - `simulate.test.js` 不再断言 `Visual Novel`，改为断言 `fa-book-open`、`Visual Novel`、`data-vnm-magic-entry="1"`。
   - 增加旧 `[data-igs-magic-entry]` 清理测试。

### 2. 新增原版消息正文抽取模块

1. 新增 `app/src/scene/message-source.js`。
2. 从原版迁移并模块化这些能力：
   - `getVisibleMessageTextFromElement(mesElement)`
   - `cleanNarrativeSource(text)`
   - `buildTagFilteredTextSource(raw, filter)`
   - `buildBridgeImageSource(raw, filter)`
   - `applyVisualNovelBodyFormat(raw, rule)`
   - `buildVisualNovelTextPayload(raw, options)`
3. 清洗规则必须包含原版移除列表：
   - DOM 移除：`script,style,iframe,button,[role="button"],[data-vnm-internal-reader],.vnm-img-ph,.mes_buttons,.extraMesButtons`
   - 字符串移除：`<style...>...</style>`、`<script...>...</script>`、`<iframe...>...</iframe>`、`<image...>`、`image###...###`、所有 HTML tag。
4. 输出结构固定：
   - `raw`
   - `visibleText`
   - `textSource`
   - `formattedText`
   - `formattedRaw`
   - `imageSource`
   - `sourceKind`
   - `warnings`
   - `errors`

### 3. 改造 host adapter 的取消息策略

1. 修改 `app/src/host/tavern-helper-adapter.js`：
   - `normalizeMessage()` 保留 `raw`，新增 `rawHtml`、`visibleText`、`sourceKind`。
   - 从 TavernHelper message 取字段时按优先级读取：`text`、`message`、`content`、`mes`，但如果值明显是宿主 UI HTML，先进入清洗管线，不直接作为 reader text。
   - `getCurrentMessage()` 不再盲取 `messages[messages.length - 1]`，需要过滤 `is_system/is_hidden`，并优先选择最近 AI/非用户消息；字段不足时保留现有 fallback。
2. 增加 DOM fallback：
   - 如果可访问 `document`，扫描 `#chat .mes` 或 `.mes`。
   - 用原版 `isScannableMessage` 思路排除系统/隐藏消息。
   - 从 `.mes_text` 抽取 `visibleText`。
3. 保证 fake TavernHelper 环境下仍可只用对象 message 运行测试。

### 4. 让 VN 阅读器默认走原版正文管线

1. 修改 `app/src/core/bootstrap.js`：
   - 普通 `refresh()` 可继续支持现有预设管线。
   - 但 `openLatestAvailable()` / `openViewerFromMessage()` 进入 VN reader 时，必须构造 Visual Novel payload，使用 `message-source.js` 的 `formattedText/formattedRaw`。
2. 修改 `app/src/api/visual-novel-compat.js`：
   - `openViewerFromMessage()` 获取 message 后，传入原始 message、`messageId`、reader mode，并确保 reader host 得到 `formattedText/formattedRaw/visibleText`。
   - `openLatestAvailable()` 也走同一条 payload 构造函数。
3. 修改 `app/src/visual/visual-novel-ui/reader-host.js`：
