# v0.11.0 情绪词分类系统与注入深度修正策划书

## 背景

参考 `_inbox/酒馆助手脚本-对话渲染系统 v7.1.json` 的情绪词分类系统。
对话渲染系统用一套固定 8 组情绪词库（`DEFAULT_MOOD_GROUPS`），通过
`{{mood_groups}}` 占位符注入约束 AI 只能从池里选词，并按"组"取差分头像。

IGS 现状：`[igs-char:角色名|情绪|对白]` 已有"情绪"字段，且 `lookupAssetValue`
已按情绪查角色立绘差分。但两个缺口：

1. 情绪词不受控——prompt 只写"情绪不得省略"，没给词池，AI 自由发挥，
   立绘差分配置命中率低。
2. 没有归约——`lookupAssetValue` 拿 mood 直接当 key 查，缺"欣喜→喜悦组"
   这层，差分图必须穷举细分词。

## 注入深度问题（codex 历史复盘）

git 历史还原：
- v0.4.0 codex 照搬对话渲染系统：`injectPrompts([{position:'in_chat',depth:0}])`。
- `6b7b63a` 改 `position:'in_prompt'`，commit 称"被截断"。
- `36e35a9`(v0.4.7) 把 `setExtensionPrompt` 提为最优先 + 回读验证，
  injectPrompts 退成 `position:'none'` 哑路径；位置定为 `IN_PROMPT(0)`。

根因：**不是参数写错，是 API 选择错配**。对话渲染系统调裸全局 `injectPrompts`
（JS-Slash-Runner 注入到脚本作用域），它本身跑在那个作用域里所以能用；
IGS 是 loader 远程 bundle 以 `<script>` 注入，拿不到那个裸函数，injectPrompts
路径静默失效。v0.4.0~6b7b63a 调完不验证，"看起来注入实际没进"被误读成"截断"。
v0.4.7 换 `setExtensionPrompt`（走 `SillyTavern.getContext()`，IGS 能稳定拿到）
才第一次真注入成功，但位置定成 `IN_PROMPT` 偏前，导致长上下文里 AI 忽视。

修正方向：**API 保持 IGS 可靠的 `setExtensionPrompt` + 回读验证不动，
只把 position 从 `IN_PROMPT(0)` 改成 `IN_CHAT(1)`、depth 保持 0**，
拿到对话渲染系统"贴在对话末尾压过一切"的效果，且不重蹈 injectPrompts 覆辙。
另补监听 `CHAT_CHANGED` 切聊天重注入（in_chat 注入只对当前聊天有效）。

注："注入到预设思维链"经全文件核查不存在——对话渲染系统无任何 preset/
promptManager/reasoning 代码，其效果就是 in_chat depth=0，本方案已等效实现。

## 实施清单

### 1. 新增 `app/src/scene/mood-groups.js`
- `DEFAULT_MOOD_GROUPS`：8 组 `{label, words[]}`（去掉 color，只用于立绘差分）。
- `resolveMoodGroup(word, groups)`：word===label 或 words 含 word → 返回 label，否则 null。
- `buildMoodGroupsText(groups)`：渲染成 `喜悦组：开心、欢喜…` 多行文本。
- `MOOD_GROUPS_PLACEHOLDER = '{{mood_groups}}'`。

### 2. 存储与归一化
- `bridge.sceneAssets.moodGroups` 新增字段，`normalizeSceneAssets` 默认填 `DEFAULT_MOOD_GROUPS`，
  校验为数组、每项 `{label:string, words:string[]}`。

### 3. 注入提示词占位符
- `DEFAULT_SCENE_PROMPT_RULE` 加情绪约束段，含 `{{mood_groups}}` 占位符。
- 注入前（bootstrap `syncSceneAssetsInjection`）若 promptRule 含占位符，
  用 `buildMoodGroupsText(sceneAssets.moodGroups)` 替换再注入；用户删占位符则不替换不报错。

### 4. 立绘差分查表三级兜底（`scene-directives.js` `lookupAssetValue`）
- 改签名为 `lookupAssetValue(record, requestedKey, moodGroups)`：
  1. `record[requestedKey]` 精确命中（兼容老的自由命名槽）。
  2. `resolveMoodGroup(requestedKey, moodGroups)` → 组名 → `record[组名]`。
  3. `record['默认']`。
- `lookupSceneAssetUrls` 透传 `sceneAssets.moodGroups`。

### 5. 注入深度修正（`prompt-injector.js`）
- `EXTENSION_PROMPT_IN_PROMPT(0)` 保留，新增 `EXTENSION_PROMPT_IN_CHAT(1)`。
- `setExtensionPrompt` 的 position 改用 `IN_CHAT(1)`，depth 0，role SYSTEM，验证逻辑保留。
- `verifyContextPrompt` 的 position 断言同步改 `IN_CHAT(1)`。
- bootstrap 监听 `CHAT_CHANGED` 重注入，destroy 时解绑。

### 6. UI（场景 tab 角色立绘卡片内）
- 角色立绘卡片标题下新增可折叠区（`<details>`，默认折叠）「情绪词库（全局）」：
  增删改组、增删改词、恢复默认词库。
- 每个角色行 `+` 旁加「一键导入情绪组名」按钮：按当前 moodGroups 的组名
  在该角色下新建空槽（已存在的组名跳过）。
- 新增 actions：`mood-add-group` / `mood-remove-group:组名` / `mood-rename-group:组名` /
  `mood-add-word:组名` / `mood-remove-word:组名:词` / `reset-mood-groups` /
  `scene-import-mood-slots:角色名`。词级校验照搬对话渲染系统：跨组去重、每组至少留 1 词。

### 7. 测试 + 版本号
- 单测：`resolveMoodGroup` 归约、`buildMoodGroupsText` 渲染、占位符替换、
  三级兜底（细分词→组槽命中 + 自定义槽精确命中两条）、注入 position=1。
- 版本号 5 处同步 v0.11.0（package.json / bootstrap.js / gate-contract.test.js /
  simulate.test.js / README.md）。
- 跑 `npm run gate`。

## 明确不做
- 不染气泡、不显 emoji、不动 v0.10.3 台词/旁白/心里话渲染。
- 不碰 SillyTavern 预设/promptManager（侵入式、易与用户预设冲突）。
- 组只服务立绘差分。
