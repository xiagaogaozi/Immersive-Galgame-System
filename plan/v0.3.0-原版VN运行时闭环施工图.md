# v0.3.0 原版VN运行时闭环施工图

## 目标

把 `app/src/visual/visual-novel-ui/reader-host.js` 从“仅保留原版 DOM 壳”补齐为“原版 VN 运行时行为等价层”，优先收口用户已经确认的高优先级问题：

1. `pc / mobile / web / fullscreen` 四模式运行时不等价。
2. 隐藏后无法恢复，黑屏只剩背景。
3. 正文因空字符串 fallback 丢失，阅读器打开后正文为空。
4. `toast`、发送状态占位、键盘快捷键没有回到原版行为。
5. 现有 `gate` 无法卡住这些退化。

## 实施范围

- `app/src/visual/visual-novel-ui/reader-host.js`
- `app/src/visual/visual-novel-ui/original-reader-source.js`
- `app/fixtures/visual-novel-ui/original-reader-snapshot.json`
- `app/tests/unit.test.js`
- `app/tests/simulate.test.js`
- `README.md`

## 实施步骤

### 1. Reader 运行时补齐

- 保留原版选择器与按钮结构不变。
- 在 reader host 中补齐：
  - `pc` 浮窗固定设计尺寸 `900x540`
  - `mobile` 浮窗固定设计尺寸 `480x680`
  - `web` 模式锁定 `body/html` 滚动，并跟随 `visualViewport.height`
  - `fullscreen` 模式调用 `requestFullscreen`，监听 `fullscreenchange`
- 关闭阅读器时统一回收：
  - `visualViewport` 监听器
  - `resize / orientationchange / fullscreenchange`
  - `body/html` 滚动锁
  - toast 定时器

### 2. 正文与隐藏恢复修复

- 新增“跳过空字符串和宿主 UI HTML”的正文 fallback。
- 背景点击层恢复原版行为：
  - 未隐藏时点击背景层翻到下一段
  - 已隐藏时点击背景层恢复对话框
- 键盘恢复原版：
  - `Escape` 关闭
  - `ArrowLeft` 上一段
  - `ArrowRight / Space` 下一段
  - `H` 隐藏/恢复

### 3. 原版 DOM 契约补全

- 补回 `#vnm-send-status`
- 补回 spinner 样式
- 保持：
  - `#vnm-overlay`
  - `#vnm-unified-settings`
  - `.vnm-*`
  - 单一魔法棒书本入口

### 4. 模拟验收闸门

- 新增单元/模拟闸门：
  - 空正文 fallback
  - `pc/mobile` 浮窗几何
  - `web` 滚动锁定与恢复
  - `fullscreen` 请求与退出关闭
  - 隐藏恢复
  - toast 边界反馈

## 验收标准

1. `npm run test` 通过。
2. `npm run simulate` 通过。
3. `npm run gate` 通过。
4. `npm run build:loader` 后 `loader/igs-loader.json.content === loader/igs-loader.js`。
5. 项目版本提升到 `v0.3.0`，并在项目独立 GitHub 仓库提交、推送、打标签。
