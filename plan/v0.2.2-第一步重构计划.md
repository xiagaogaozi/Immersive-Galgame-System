## Summary

- 现在可以开始重构，但第一步不直接迁移完整阅读器 UI，也不直接改原版 `Visual Novel` 本体。

## 施工步骤

1. 建立迁移文档
   - 新增 `docs/VISUAL_NOVEL_MIGRATION.md`。
   - 写明原版参考源当前来自 `projects/Visual Novel.zip`，不是当前空的 `projects/Visual Novel/` 解压目录。
   - 列出 v9.6.6 的公开 API、旧存储键和第一阶段不迁移范围。

2. 新增旧配置只读适配
   - 在 `app/src/storage/legacy-visual-novel.js` 中实现：
     - `LEGACY_VN_KEYS.bridge = 'vn_visual_novel_bridge_config'`
     - `LEGACY_VN_KEYS.readerPrefix = 'vn-reader-settings-v9-'`
     - `LEGACY_VN_KEYS.displayMode = 'vn-display-mode'`
     - `readLegacyVisualNovelSettings(storageLike, mode)`
     - `normalizeLegacyVisualNovelSettings(raw)`
   - 默认只读，不自动写回 `vn:*`，避免污染用户现有配置。
   - 失败返回 `{ ok:false, reason:'invalid-legacy-json' }`，不 throw。

3. 拆出 Visual Novel 兼容 API 层
   - 新增 `app/src/api/visual-novel-compat.js`。
   - 导出 `createVisualNovelCompatApi(app)`。
   - 将以下方法归入兼容层：
     - `openSettings(options)`：当前阶段返回 `{ ok:false, reason:'settings-ui-not-mounted', options }`，并通过 `app.events.emit('vn:settings-requested', options)` 留出未来 UI 入口。
     - `getConfig()`：返回 `app.getState().config` 的浅拷贝。
     - `getUnifiedSettings(options)`：返回 VN 兼容形状 `{ version, bridge, imageApi, readerMode, readerSettings }`。
     - `generateImage(request, options)`：当前阶段返回 `{ ok:false, reason:'provider-not-enabled', request, options }`，未来接 `generated-images` provider。
     - `openViewerFromMessage(messageId, mode)`：第一阶段用 fake host / fixture 测试，最小行为是调用 `app.refresh({ messageId, viewerMode: mode })` 或返回明确不可用原因，不允许静默成功。
     - `openLatestAvailable(mode, options)`：第一阶段委托当前 host 的最后消息读取路径，成功时返回 `refresh` 结果。
   - `destroy()` 继续使用现有 `app.destroy`。

4. 连接 `public-api`
   - `createPublicApi(app)` 内调用 `createVisualNovelCompatApi(app)`。
   - 将兼容方法 spread 到公开 API 上。
   - 不新增 `window.VisualNovelBridge`，避免和 `api/CONTRACT.md` 冲突；如后续实测必须支持旧全局，再单独开计划。

5. 增加 fixtures
   - `legacy-storage.json`：包含旧 bridge config、`pc` readerSettings、`vn-display-mode`。
   - `api-contract.json`：列出第一阶段必须存在的方法名与返回形状字段。
   - `vn-message.json`：用短正文覆盖说话人、情绪、时间、天气、地点、生图段，不复制原版长源码。

6. 增加 gate 测试
   - `gate:visual-novel-compat:api-shape`
     - 断言 `window.VN` 上存在 VN 兼容方法。
     - 断言 `getUnifiedSettings({ mode:'pc' })` 返回 `version`、`bridge`、`imageApi`、`readerMode`、`readerSettings`。
   - `gate:visual-novel-compat:legacy-storage`
     - 断言旧键可被读取和归一化。
     - 断言坏 JSON 返回错误对象而不是抛出。
   - `gate:simulation:visual-novel-open-latest`
     - 使用 fake host 消息调用 `openLatestAvailable('pc')`，断言走到 scene/visual 最小闭环。

7. 更新文档与记录
   - `README.md` 新增 `v0.2.2` 更新日志。
   - `docs/PACKAGING_WORKFLOW.md` 可补充 `docs/VISUAL_NOVEL_MIGRATION.md` 为迁移必读。
   - 不写 `奶龙工具箱/CHANGELOG.md`。

## 数据流/接口契约

```text
Visual Novel v9.6.6 参考源
  -> docs/VISUAL_NOVEL_MIGRATION.md 记录兼容面
  -> app/fixtures/visual-novel/* 固化短样例
  -> app/src/storage/legacy-visual-novel.js 读取旧配置
  -> app/src/api/visual-novel-compat.js 提供兼容 API
  -> app/src/api/public-api.js 挂到 window.VN
  -> app/tests/* 纳入 npm run gate
```

公开 API 第一阶段兼容输出：

```js
VN.getConfig();
VN.getUnifiedSettings({ mode: 'pc' });
VN.openSettings({ source: 'reader', tab: 'basic', mode: 'pc' });
VN.openViewerFromMessage(messageId, 'pc');
VN.openLatestAvailable('pc');
VN.generateImage({ prompt: '...' });
VN.destroy();
```

旧配置输入：

```text
vn_visual_novel_bridge_config
vn-reader-settings-v9-pc
vn-reader-settings-v9-mobile
vn-reader-settings-v9-web
vn-reader-settings-v9-fullscreen
vn-display-mode
```

VN 输出仍使用：

```text
vn:*
```

第一步只读旧键，不自动写入新键。

## 通过标准

- `npm run gate` 全部通过。
- `window.VN` 和 `window.VisualNovel` 保持存在。
- `VN.getUnifiedSettings({ mode:'pc' })` 返回 VN 兼容字段，不抛错。
- `VN.openSettings()`、`VN.generateImage()` 在未实现 UI/provider 时返回明确 `{ ok:false, reason }`。
- 旧 `vn_*` 配置读取失败不会中断启动。
- `openLatestAvailable('pc')` 在 fake host 下可触发最小 scene/visual 刷新，或在无消息时返回明确 `{ ok:false, reason:'no-message' }`。
- 不新增 `project.json/latest/archive/tavern helper` 发布壳。
- 不复制完整 `Visual Novel` 大源码到 VN；只保留短 fixtures 和迁移文档。

- 不实现设置面板。

## 残余风险

- 当前 `projects/Visual Novel/` 解压目录为空；若后续要做逐函数迁移，需先恢复 source-first 目录或用只读脚本从 `Visual Novel.zip` 抽取参考。
- `openViewerFromMessage()` 的完整行为依赖按楼层读取消息，目前 host 适配层只有最后消息读取；第一步只能做最小兼容，完整上一轮/下一轮需后续重构 host。
- `generateImage()` 第一阶段不接真实 provider；生图体验不会因此恢复。
- `openSettings()` 第一阶段不显示 UI；只建立事件和返回契约。
