# v0.2.6 Visual Novel 原版 UI 等价迁移施工图

## Summary

- 本轮目标是把 `projects/Visual Novel` 的原版阅读器 UI 与统一设置面板按等价体验迁移到 IGS，而不是继续停留在 `stage model` / 槽位骨架。
- 等价迁移的核心验收是：原版阅读器覆盖层、工具栏、输入区、显示设置、统一设置面板、`基础 / 正文替换 / 图像 / 阅读器` 四个 tab、移动端/桌面端/网页全屏/全屏模式均具备原行为和原视觉结构。
- 本轮不修改 `projects/Visual Novel` 本体，不走角色卡流程，不使用奶龙工具箱 `pack-project` / `verify-project` 作为验收；验收仍以 IGS 项目自己的 `npm run gate` 和模拟 UI 快照测试为准。

## UI 预期

### 用户可见目标

1. 从酒馆魔法棒菜单触发 IGS 时，应直接打开原版 Visual Novel 黑底阅读器体验，不出现简化阅读器、占位卡片或只展示 stage model 的调试结构。
2. 阅读器覆盖层保留原版核心 DOM 语义：`#vnm-overlay`、`#vnm-bg`、`#vnm-bg-blur`、`#vnm-click-layer`、`.vnm-dialog`、`.vnm-ctrl-bar`、`.vnm-controls`、`#vnm-input`、`#vnm-send-btn`、`#vnm-settings`、`#vnm-toast`。
3. `pc` / `mobile` / `web` / `fullscreen` 四类模式继续存在，且 `pc` / `mobile` 保留浮窗尺寸、拖动边界、工具栏外露和输入区不裁切。
4. 设置面板打开后应是原版统一设置面板：遮罩 `#vnm-unified-settings`、`.vnm-settings-shell`、顶部标题/版本/关闭、横向 tabs、可滚动 body；不存在底部 `保存 / 取消` 操作栏，字段修改即时保存。
5. 设置面板 tab 必须包含 `基础`、`正文替换`、`图像`、`阅读器` 四项；`正文替换` 页必须保留 `标签解析` 在上、`正文格式化` 在下；`图像` 页必须保留外部插图扩展与 VN 内置 NAI API 的模式切换；`阅读器` 页必须保留原版 reader settings 字段。
6. 原版旧存储 `vnm_visual_novel_bridge_config`、`vnm-reader-settings-v9-pc/mobile/web/fullscreen` 的读取兼容继续生效；新 IGS 状态可以镜像，但不能丢失旧配置意义。

### 不等价即失败的情况

- 用 `.igs-stage` 的简单对话层替代原版 `#vnm-overlay` / `.vnm-dialog`。
- 只保留 `设置` 一个空壳，未迁移四个原版 tab。
- 删除 `正文替换` 页的测试当前楼层、预览区或恢复默认按钮。
- 删除 `图像` 页的外部扩展检测、模型字段、测试生成/检测插件反馈。
- 删除阅读器内显示设置弹层 `#vnm-settings` 或常驻按钮配置。
- 删除 `pc` / `mobile` 浮窗模式，只保留全屏。
- 让 `openSettings()` 继续返回 `settings-ui-not-mounted`。

## 数据流 / 接口契约

```text
酒馆魔法棒入口 / IGS.openLatestAvailable()
  -> hostAdapter.getCurrentMessage() 或 getMessageById()
  -> app.refresh({ messageId, viewerMode })
  -> text-pipeline + scene resolver + stage model
  -> build Visual Novel reader payload
  -> openVisualNovelReader(payload, { mode })
  -> 原版 #vnm-overlay / .vnm-dialog / .vnm-controls
  -> 输入框 Enter / 发送按钮
  -> hostAdapter.typeIntoInputAndSend(text)
```

```text
IGS.openSettings({ tab, mode })
  -> openVisualNovelSettings()
  -> read legacy + IGS config
  -> render shell.html + tabs/*.html + style.css
  -> field input/change/click
  -> saveUnifiedSettings-compatible adapter
  -> update localStorage-compatible storage
  -> emit reader settings event / update reader state
```
