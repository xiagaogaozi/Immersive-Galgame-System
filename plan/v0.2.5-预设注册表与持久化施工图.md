- 按本计划实施时属于 R3 跨模块运行时/registry/storage/presets/core/api 修改，需要运行 `npm run gate`。

## 运行预期

实施后，文本 preset 不再只能通过 `refresh({ textFilterPreset, textFormatPreset, sceneRegexPreset })` 临时传入：

- `dispatchImportBundle()` 可以继续做白名单分发，但三类文本 preset 应由新的 `PresetRegistry` 接收、校验并写入持久化 snapshot。
- 公开 API 的 `VN.api.textFilterPresets`、`VN.api.textFormatPresets`、`VN.api.sceneRegexPresets` 保留现有 `register/unregister/get/list`，并新增 `setCurrent(id)`、`getCurrent()`、`export(id)`、`exportAll()`。
- `setCurrent(id)` 只能指向已经成功保存且校验通过的 preset；坏正则 preset 可以返回错误并进入 rejected/invalid 结果，但不能覆盖当前启用项。
- `bootstrapVN()` 启动时从 preset store hydrate registry；`refresh()` 在没有 context 覆盖项时，从 registry 当前启用项取 `textFilterPreset`、`textFormatPreset`、`sceneRegexPreset`。
- `state.config` 仍可作为兼容输入；优先级建议为：`context` 显式传入 > registry 当前启用项 > `state.config` > 默认解析。
- 导出结果保持 `igs_preset_v1` 单项格式或 `vn-import-bundle` 合集格式，不夹带真实 API key、私有聊天记录或长 prompt。

1. 定义 preset 类型映射

   - 导出：
     - `PRESET_TYPES`
     - `TEXT_PRESET_TYPES`
     - `PRESET_TYPE_TO_API_GROUP`
     - `PRESET_TYPE_TO_CONFIG_KEY`
     - `normalizePresetManifest(preset)`
     - `validatePresetManifest(preset)`
     - `validatePresetByType(preset)`
   - 必须覆盖当前已登记 preset 类型，至少包含：
     - `scene-regex-preset`
     - `text-filter-preset`
     - `text-format-preset`
     - `prompt-preset`
     - `image-provider-preset`
     - `image-request-builder-preset`
     - `workflow-preset`
     - `theme-preset`
     - `css-preset`
     - `ui-skin-preset`
     - `ui-layout-preset`
     - `background-rule-preset`
     - `choice-parser-preset`
     - `visual-mode-preset`
   - `validatePresetByType()` 对三类文本 preset 复用 `text-presets.js` 的 validator；其余 preset 本轮只做 manifest 基础校验，后续按能力补 type-specific validator。

2. 建立 preset store
   - 新增 `app/src/storage/preset-store.js`。
   - 导出：
     - `PRESET_STORE_KEY = 'vn:preset-registry:v1'`
     - `createPresetStore(storageLike, options = {})`
     - `createMemoryStorage(initialData = {})`
     - `readPresetSnapshot(storageLike)`
     - `writePresetSnapshot(storageLike, snapshot)`
   - `storageLike` 支持 `getItem/setItem/removeItem`；方法可异步返回，方便后续替换 IndexedDB。
   - snapshot 建议形状：

```js
{
    version: 1,
    current: {
        'text-filter-preset': 'preset.text-filter.content-only',
        'text-format-preset': 'preset.text-format.bubble-line',
        'scene-regex-preset': 'preset.scene-regex.stage-fields',
    },
    items: {
        'text-filter-preset': {
            'preset.text-filter.content-only': { ...preset },
        },
    },
    drafts: {},
    updatedAt: '2026-06-12T00:00:00.000Z',
}
```

3. 建立 `PresetRegistry`
   - 新增 `app/src/presets/preset-registry.js`。
   - 导出：
     - `createPresetRegistry(options = {})`
     - `createPresetGroup(registry, type)`
     - `importPresetBundle(registry, bundle, options = {})`
   - `createPresetRegistry()` 方法建议：
     - `hydrate()`
     - `snapshot()`
     - `register(preset, options = {})`
     - `unregister(type, id)`
     - `get(type, id)`
     - `list(type)`
     - `setCurrent(type, id)`
     - `getCurrent(type)`
     - `getCurrentData(type)`
     - `exportPreset(type, id)`
     - `exportGroup(type)`
     - `importBundle(bundle, options = {})`
   - `register()` 返回形状建议：

```js
{
    ok: true,
    item,
    normalized,
    replaced: false,
    currentChanged: false,
    errors: [],
}
```

   - `register()` 在校验失败时必须返回 `ok:false`，不得保存到 active items，不得覆盖 current。
   - 可选策略：校验失败的 preset 本轮不进 `items`，只在 import result 的 `rejected` 中返回；不要新增 UI 草稿行为。
   - `setCurrent()` 只能接受已存在且校验通过的 `id`；失败返回 `{ ok:false, reason:'missing-preset' }` 或 `{ ok:false, reason:'invalid-preset' }`。

4. 连接导入分发
   - 保留 `app/src/registry/import-dispatcher.js` 的纯白名单职责。
   - 在 `PresetRegistry.importBundle()` 内调用 `dispatchImportBundle(bundle, handlers)`，handlers 只处理 preset 类型。
   - 对于非 preset 但 allowed 的类型，保持 accepted/rejected 结果可解释：
     - 本轮可返回 `rejected: [{ item, reason:'no-preset-handler' }]`，避免误存 Mod 或 Pack。
   - 对于 forbidden 类型仍沿用 `dispatchImportBundle()` 的 `unsupported-type` 行为。

5. 连接公开 API
   - 修改 `app/src/api/public-api.js`。
   - 保留旧的 group 方法：
     - `register(item)`
     - `unregister(id)`
     - `get(id)`
     - `list()`
   - 对 preset-backed group 新增：
     - `setCurrent(id)`
     - `getCurrent()`
     - `export(id)`
     - `exportAll()`
   - `sceneRegexPresets`、`textFilterPresets`、`textFormatPresets` 必须走 `PresetRegistry`；其他 group 暂可保留内存 Map，直到对应能力页施工。
   - 不提供 `registerAnything`；不新增全局总 Mod 管理 API。

6. 连接 `bootstrapVN()`
   - 修改 `app/src/core/bootstrap.js`。
   - 初始化时创建：
     - `const presetStore = options.presetStore || createPresetStore(storageLike)`
     - `const presetRegistry = options.presetRegistry || createPresetRegistry({ store: presetStore })`
   - 启动时执行 `await` 可能会影响当前同步 `bootstrapVN()` 返回值；本轮建议不要把 `bootstrapVN()` 改成 async。
   - proposed default：`createPresetRegistry()` 构造时同步读取 localStorage-compatible snapshot；后续真实 IndexedDB 再单独引入 async hydration。
   - `app` 内部暴露：
     - `presetRegistry`

   - `getState()` 增加 `presetRegistry: presetRegistry.snapshot()` 或 `presets: presetRegistry.snapshot()`，避免暴露内部可变 Map。
   - `refresh()` preset 优先级：
     - `context.textFilterPreset`
     - `presetRegistry.getCurrentData('text-filter-preset')`
     - `state.config.textFilterPreset`
     - 默认空


- 导出 bundle 不包含真实 API key、私有聊天记录、真实 shujuku 数据或长 prompt。
- 本轮不新增真实设置 UI、不挂载 `window.VisualNovelBridge`、不修改 `projects/Visual Novel` 本体、不执行真实酒馆实机验真。

## 不做范围

- 不实现正则与正文页 UI。
- 不实现能力页下拉组件和编辑表单。
- 不实现真实 IndexedDB 大资源存储、Blob URL 生命周期或容量统计。
- 不实现 Mod 代码加载、权限系统或总管理页。
- 不实现 Pack 资源持久化。
- 不接真实 shujuku 数据读取。
- 不接真实 provider 或生图轮询。
- 不打包发布 loader，除非后续实现中实际修改发布链路。

## 残余风险

- 本轮建议先用 localStorage-compatible / memory storage 固定 registry 契约；真实 IndexedDB adapter 后续仍需单独施工，尤其要处理浏览器异步启动和迁移失败回滚。
- 当前只有文本 preset 有 type-specific validator；其他 preset 本轮若进入基础 manifest 校验，后续各能力页需要补专属 schema 与坏数据测试。
- 公开 API 增加 `setCurrent/getCurrent/export/exportAll` 后，后续 UI 应复用同一 registry group，避免能力页各自维护第二套列表。
- 如果后续把 `bootstrapVN()` 改成 async 来等 IndexedDB hydrate，需要重新设计 loader 启动时序；v0.2.5 先不触碰这个风险点。
