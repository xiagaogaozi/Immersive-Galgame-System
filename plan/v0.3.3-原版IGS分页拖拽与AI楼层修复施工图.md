# v0.3.3 原版 IGS 分页拖拽与 AI 楼层修复施工图

## Summary

- 本施工图目标是修复当前 IGS 与原版 Immersive Galgame System 在阅读分页、PC/手机浮窗拖拽、上一轮/下一轮 AI-only 楼层扫描、长文本滚动和四模式选择器溢出上的差异。
- 本轮计划不实施代码；实现轮需要按本文件重新核对源码后开工。
- 本轮边界是 `projects/<当前本地项目文件夹>/` 独立 app；不走角色卡流程，不接入奶龙工具箱 `project.json / latest / archive / tavern helper` 发布壳。

## 已核对落点

1. `reader-host.js / buildTextSegments()` 当前分页规则偏离原版，必须作为第一修复点。
2. `reader-host.js / applyInlineReaderRuntime()` 当前有 viewport clamp，但缺少原版 `installFloatingDrag()` 行为。
3. `reader-host.js / applyReaderSnapshotToDom()` 当前 click layer 直接翻页，需要接入拖拽 suppress。
4. `original-reader-source.js / ORIGINAL_READER_STYLE_TEXT` 当前缺少原版浮窗文本滚动和拖拽状态样式。
5. `tavern-helper-adapter.js / isUserMessage()` 当前用户层识别不覆盖字符串、数字、`role:'user'` 或 DOM 属性。
6. `bootstrap.js / resolveAdjacentMessage()` fallback 不做 AI-only 过滤。
7. `simulate.test.js` 中已有测试会锁住错误分页行为，必须同步改成原版规则。

## UI 施工图

- 阅读器仍挂载到 `#igs-overlay`，保留 `.igs-mode-pc/.igs-mode-mobile/.igs-mode-web/.igs-mode-fullscreen` 与 `.igs-floating/.igs-floating-mobile`。
- PC/手机模式浮窗：
  - `container`: `#igs-overlay`
  - `anchor`: viewport 底部居中，拖拽后改为用户拖动位置
  - `placement`: overlay fixed
  - `size`: PC `900px x 540px` 上限；手机 `480px x 680px` 上限；受 viewport clamp 限制
  - `spacing`: viewport clamp 边距 `8px`，初始 bottom gap `24px`
  - `overflow`: overlay `overflow:hidden`；`.igs-dialog overflow:visible`；`.igs-text overflow-y:auto`
  - `source`: 原版 `original-reader.js:1154-1166`、`1503-1558`
- 文本分页：
  - 每个 segment 等于清洗后的一行/一段，即原版 `cleaned.split(/\n+/)` 后的非空项。
  - 一行里有多个句号、问号、感叹号时仍然是一页。
  - 单换行分隔的多段必须拆成多页。
- 长文本布局：
  - `.igs-progress` 固定在文本上方，不被正文挤走。
  - `.igs-text` 在浮窗可用高度内滚动。
  - `.igs-controls` 固定在底部，不被正文覆盖。
- 四模式选择器：
  - 保留 `.igs-segmented`、`.igs-segmented-btn`、`.igs-segmented-btn-icon`、`.igs-segmented-btn-label` 和四个文案 `电脑 / 手机 / 网页全屏 / 全屏`。
  - 宽度不足时 `.igs-segmented-btn-label` 使用 `overflow:hidden; text-overflow:ellipsis; white-space:nowrap; display:block; max-width:100%`。
  - `.igs-segmented-btn` 必须保留 `min-width:0`，并补 `overflow:hidden`，防止图标+文本撑破按钮。

## 函数级施工步骤

1. `app/src/visual/igs-ui/reader-host.js`
   - 将 `buildTextSegments(text)` 改为原版语义：
     - 保留 `\n` 作为段落边界。
     - 使用 `.split(/\n+/)` 生成段落。
     - 每段执行 `.replace(/\s+/g, ' ').trim()`。
     - 删除按句尾标点拆句的逻辑。
     - 空文本返回 `['']`。
   - 在 `applyReaderSnapshotToDom()` 的 click layer 点击分支前检查 `current.dragSuppressClick` 或 runtime suppress 标志；为真时清掉标志并直接 return。
   - 在 `applyInlineReaderRuntime(root, mode, runtime)` 中调用新的 `installFloatingDragRuntime(root, runtime)`。
   - 新增 `installFloatingDragRuntime(root, runtime)`：
     - 查找 `#igs-click-layer`。
     - pointerdown 记录 `startX/startY/startLeft/startTop/pointerId`。
     - pointermove 距离达到 `6px` 后进入拖拽，给 root 加 `is-dragging`。
     - 使用 `getInlineViewportMetrics(win, doc)` 计算边界并 clamp。
     - pointerup/cancel 释放 pointer capture，移除监听，拖拽过则 suppress click `120ms`。
     - cleanup 中移除 `pointermove/pointerup/pointercancel`。
   - 避免 resize/orientation 的 clamp 立即覆盖用户拖拽位置：runtime 增加 `dragged` 标志，用户拖动后 resize 才重新 clamp 到可视区，不强制回到底部居中。

2. `app/src/visual/igs-ui/original-reader-source.js`
   - 补齐原版浮窗样式：
     - `#igs-overlay.igs-floating.is-dragging #igs-click-layer{cursor:grabbing;}`
     - `#igs-overlay.igs-floating .igs-progress{flex-shrink:0;}`
     - `#igs-overlay.igs-floating .igs-text{min-height:0;overflow-y:auto;margin-bottom:12px;flex:1 1 auto;}`
     - `#igs-overlay.igs-floating .igs-controls{flex-shrink:0;}`
   - 保持现有原版 SVG 图标常量名称与图标不变。

3. `app/src/host/tavern-helper-adapter.js`
   - 新增 `isTruthyFlag(value)`：
     - true values: `true`、`1`、`'1'`、`'true'`
   - 扩展 `isUserMessage(message)`：
     - raw 字段：`is_user`、`isUser`、`user`
     - role 字段：`message.role === 'user'`
     - DOM 字段：`message.element` 或 raw DOM 节点的 `getAttribute('is_user') === 'true'`
   - 扩展 `isSystemMessage()` 与 `isHiddenMessage()`：
     - 使用 `isTruthyFlag()` 覆盖 `is_system/isSystem/system/is_hidden/isHidden/hidden`
     - DOM `is_system="true"` 也计入系统层
   - 保持 `isTurnCandidate()` 统一使用 `!isSystem && !isHidden && !isUser`。

4. `app/src/core/bootstrap.js`
   - 修改 fallback `resolveAdjacentMessage()`：
     - 从 `hostAdapter.listMessages()` 结果中过滤 `!message.isUser && !message.isSystem && !message.isHidden`。
     - 查找当前 messageId 时允许当前消息在原列表中，但移动目标只来自过滤后的 AI turn 列表。
     - 若当前 id 不在过滤列表中，使用原列表位置向 delta 方向寻找最近 AI turn。

## 通过标准

- `buildTextSegments()` 与原版 IGS 的 `/\n+/` 分段行为一致：一段一页，不按句子自动拆页。
- PC/手机浮窗支持拖动，拖动时 class 包含 `is-dragging`，拖动后 `120ms` 内 click 不翻页。
- `prev-turn` / `next-turn` 不会打开用户层、系统层或隐藏层。
- 长文本不会覆盖或挤掉 `.igs-controls`，正文区域可滚动。
- `.igs-segmented` 四模式按钮在移动端不出现文字越界。
- `npm run gate` 和 `npm run build:loader` 通过。
- `loader/igs-loader.json` 的 `content` 与 `loader/igs-loader.js` 完全一致。
