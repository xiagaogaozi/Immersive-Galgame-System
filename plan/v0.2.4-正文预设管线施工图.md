## 运行预期

实施后，正文进入 `scene` 前会先经过一条可测试的文本管线：

- `text-filter-preset` 负责从 `<content>...</content>`、`<now_plot>...</now_plot>` 等标签块中选择正文，并排除 `thinking`、`Status_block`、`text_to_image` 等非正文块。
- `text-format-preset` 负责对过滤后的正文应用安全的正则 replacement，例如把 `@bubble:name|...|text` 转成 `[角色]：正文`。
- `scene-regex-preset` 负责从过滤/格式化后的正文中提取 `speaker`、`emotion`、`time`、`weather`、`location`、`generatedImage` 等字段。
- `parseSceneText()` 仍保留当前方括号标签和说话人前缀解析，作为默认路径和回退路径。
- 坏正则不能让 `refresh()` 抛出；必须返回明确 `ok:false` 或在 scene metadata 中记录 `errors`，同时保留可用的回退正文。
- 预设错误不得覆盖当前启用项；本轮只做运行时函数和模拟导入，不做持久化切换 UI。

## 施工步骤

1. 建立文本预设 normalizer
   - 新增 `app/src/presets/text-presets.js`。
   - 导出：
     - `DEFAULT_TEXT_FILTER_PRESET`
     - `DEFAULT_TEXT_FORMAT_PRESET`
     - `normalizeTextFilterPreset(presetOrData)`
     - `normalizeTextFormatPreset(presetOrData)`
     - `normalizeSceneRegexPreset(presetOrData)`
     - `validateTextFormatPreset(presetOrData)`
     - `validateSceneRegexPreset(presetOrData)`
   - `text-filter-preset` 默认值参考原版事实，但以 IGS 命名保存：
     - `enabled: true`
     - `stripHtmlComments: true`
     - `allowUntaggedFallback: false`
     - `textIncludeTags: "content"`
     - `textExcludeTags: "thinking\nSubtext_think\nStatus_block\ntext_to_image\nparallel_world\naftertalk"`
     - `imageIncludeTags: "image\ntext_to_image"`
     - `imageExcludeTags: ""`
   - `text-format-preset` 默认值：
     - `enabled: false`
     - `pattern: ""`
     - `flags: ""`
     - `replacement: ""`
   - `scene-regex-preset` 默认不强制启用；传入时按字段逐项编译。

2. 建立正文预处理管线
   - 新增 `app/src/scene/text-pipeline.js`。
   - 导出：
     - `parseTagList(text)`
     - `extractTagBlocks(raw, tags, options)`
     - `removeTagBlocks(raw, tags)`
     - `buildFilteredTextSource(raw, filterPreset, options)`
     - `applyTextFormatPreset(text, formatPreset)`
     - `applySceneRegexPreset(text, regexPreset)`
     - `runTextPipeline(raw, options = {})`
   - `runTextPipeline()` 返回形状建议：

```js
{
    ok: true,
    raw,
    textSource,
    formattedText,
    imageSource,
    scenePatch,
    sourceKind,
    formatSourceKind,
    expectedTags,
    warnings: [],
    errors: [],
}
```

3. 扩展 `parseSceneText()`
   - 保持 `parseSceneText(text, options)` 函数名不变，避免破坏现有调用。
   - 当 `options.textFilterPreset` / `options.textFormatPreset` / `options.sceneRegexPreset` 存在时，先执行 `runTextPipeline()`。
   - 当前方括号标签解析继续作为默认解析器；scene regex 输出优先级高于说话人前缀，但低于显式方括号标签，除非 preset 明确 `overrideExplicitTags: true`。
   - 在返回 scene 中增加非破坏性 metadata：
     - `textSource`
     - `formattedText`
     - `sourceKind`
     - `formatSourceKind`
     - `textPipelineWarnings`
     - `textPipelineErrors`
   - `generatedImage` 可继续支持 `[生图: ...]`；scene regex 提取到的 `generatedImage` 需要归一化为 `{ source:'scene-regex-preset', value }`。

4. 连接 `bootstrapIGS()`
   - `refresh(context = {})` 调用 `parseSceneText()` 时传入：
     - `textFilterPreset: context.textFilterPreset || state.config.textFilterPreset`
     - `textFormatPreset: context.textFormatPreset || state.config.textFormatPreset`
     - `sceneRegexPreset: context.sceneRegexPreset || state.config.sceneRegexPreset`
   - 不在 `core` 中写具体正则逻辑，`core` 只传递配置。

5. 扩展公开 API 分组
   - 修改 `app/src/api/public-api.js` 的 `API_GROUPS`，新增：
     - `textFilterPresets`
     - `textFormatPresets`
   - 保留已有 `sceneRegexPresets`。
   - 修改 `app/src/api/CONTRACT.md`，列出三类文本预设分组。

6. 增加 fixtures
   - `tagged-content-message.json`：短正文，包含 `<content>`、`<thinking>`、`<text_to_image>`、`<地点>`、`<天气>` 等短标签。
   - `text-filter-preset.json`：保留 `content`，排除非正文标签。
   - `text-format-preset.json`：用短正则把 `@bubble:玉子|开心|你好` 格式化为 `玉子：你好`。
   - `scene-regex-preset.json`：提取 `location`、`time`、`weather`、`speaker`、`emotion`。
   - `bad-text-format-preset.json`：包含非法正则，用于确认失败不崩溃。
   - 所有 fixtures 必须短小脱敏，不放真实聊天记录和长 prompt。

7. 增加 gate 测试
   - `gate:scene:text-filter-preset:extracts-content`
     - 断言 `<content>` 中正文被保留，`<thinking>` 与 `<text_to_image>` 正文不进入 `scene.text`。
   - `gate:scene:text-format-preset:applies-replacement`
     - 断言格式化 preset 能把短格式转换为可读正文。
   - `gate:scene:scene-regex-preset:extracts-fields`
     - 断言可从正文中提取 `speaker`、`emotion`、`time`、`weather`、`location`。
   - `gate:scene:text-pipeline:bad-regex-does-not-throw`
     - 断言非法正则不会让 `parseSceneText()` 或 `refresh()` 抛出；返回 scene 中有 `textPipelineErrors`。
   - `gate:import-contract:text-presets`
     - 断言 `dispatchImportBundle()` 接受三类文本预设。
   - `gate:api:public-api-exposes-text-preset-groups`
     - 断言 `IGS.api.textFilterPresets`、`IGS.api.textFormatPresets`、`IGS.api.sceneRegexPresets` 均存在。
   - `gate:simulation:text-preset-pipeline-refresh`
     - fake host message + 三类 preset 调用 `refresh()`，断言 scene 字段正确进入 visual stage dialogue。

8. 更新文档与记录
   - `README.md` 新增 `v0.2.4 - 2026-06-12` 更新日志。
   - `docs/IGS_MIGRATION.md` 增加「第三阶段：正文过滤、正文格式化、scene regex」记录。
   - `app/fixtures/README.md` 增加 `text/` 分层说明。


## 通过标准

- `npm run gate` 全部通过。
- `parseSceneText()` 在无 preset 时保持现有 `[角色: 值]` 与说话人前缀解析行为。

## 残余风险

- 原版 IGS 的文本管线包含图片缓存、DOM 图片探测和阅读器句子切分；v0.2.4 只迁移正文过滤、格式化和 scene regex，不迁移图片 DOM 探测。
- 正则 preset 如果允许用户写复杂表达式，后续真实 UI 需要加入超时、长度限制或预览保护；本轮先用短 fixtures 和非法正则测试保证不崩。
- 预设持久化仍未实现；本轮只保证运行时可传入和公开 API 可注册。
- shujuku 数据优先级还未进入 scene 合并；后续应单独开数据层计划。
