## UI 预期

实施后，阅读器最小闭环应能产出一个可被测试的 IGS 舞台模型，而不是只把 scene 传给任意 layer：

- 舞台根为 `.igs-stage`，包含 `style-contract.js` 已要求的背景、生图、角色、对话、HUD、工具栏、选项、系统层核心槽位，并在 stage model 中预留环境效果层和头像层。
- 对话层 `.igs-dialogue-layer` 能显示当前 `speaker` 与 `text`，并保留名字牌和头像的显示开关语义。
- HUD 层 `.igs-hud-layer` 内存在 `.igs-toolbar`，并能通过 `data-igs-toolbar-layout`、`data-igs-toolbar-placement` 和 `data-placement` 表达横排、竖排、位置。
- 阅读器设置进入模型后，同步为 `data-igs-*` 属性和 `--igs-*` CSS 变量；Skin CSS 不需要读业务对象即可响应布局。
- 桌面、手机竖屏、手机横屏三种布局都能通过 fixtures 模拟，不依赖真实浏览器方向锁。
- `visualMode` 继续由 `scene` 和设置决定；生图段出现时可落到 generated slot，普通文本落到 background/character/dialogue slots。

## 施工步骤

1. 建立 reader state 归一化
   - 新增 `app/src/visual/reader-state.js`。
   - 导出 `createReaderState(input = {})`。
   - 输入来源：`scene`、`visualSettings`、`readerSettings`、`layoutSettings`、`viewport`、`legacyImmersiveGalgameSystem.readerSettings`。
   - 输出至少包含：
     - `mode`: `pc` / `mobile` / `web` / `fullscreen`
     - `layout`: `desktop` / `mobile-portrait` / `mobile-landscape`
     - `toolbarLayout`: `horizontal` / `vertical`
     - `toolbarPlacement`: `top-left` / `top-right` / `top` / `bottom` / `bottom-right` / `custom`
     - `dialogueStyle`: `panel` / `subtitle` / `bubble`
     - `nameplateVisible`: boolean
     - `avatarVisible`: boolean
     - `cssVars`: 包含 `--igs-dialogue-font-size`、`--igs-dialogue-width`、`--igs-dialogue-height`、`--igs-dialogue-opacity`、`--igs-toolbar-placement-x`、`--igs-toolbar-placement-y`
     - `attributes`: 包含 `data-igs-toolbar-layout`、`data-igs-toolbar-placement`、`data-igs-dialogue-style`、`data-igs-nameplate-visible`、`data-igs-avatar-visible`
   - 默认值使用本策划书「视觉数值表」里的 `proposed default`，后续真实 DOM 接入后再实测替换。

2. 建立 stage model
   - 新增 `app/src/visual/stage-model.js`。
   - 导出 `createStageModel(scene, readerState, options = {})`。
   - 输出对象建议形状：

```js
{
    type: 'igs-stage-model',
    scene,
    readerState,
    slots: [
        '.igs-stage',
        '.igs-background-layer',
        '.igs-generated-layer',
        '.igs-character-layer',
        '.igs-dialogue-layer',
        '.igs-hud-layer',
        '.igs-toolbar',
        '.igs-choice-layer',
        '.igs-system-layer',
    ],
    attributes: {
        'data-igs-toolbar-layout': 'horizontal',
        'data-igs-toolbar-placement': 'top-right',
        'data-igs-dialogue-style': 'panel',
        'data-igs-nameplate-visible': 'true',
        'data-igs-avatar-visible': 'false',
    },
    cssVars: {
        '--igs-dialogue-font-size': '18px',
        '--igs-dialogue-width': 'min(920px, calc(100vw - 32px))',
        '--igs-dialogue-height': 'auto',
        '--igs-dialogue-opacity': '0.72',
        '--igs-toolbar-placement-x': '16px',
        '--igs-toolbar-placement-y': '16px',
    },
    layers: {
        background: { slot: '.igs-background-layer', visible: true, resource: scene.background },
        generated: { slot: '.igs-generated-layer', visible: Boolean(scene.generatedImage), resource: scene.generatedImage },
        effect: { slot: '.igs-effect-layer', visible: Boolean(scene.weather || scene.time), weather: scene.weather, time: scene.time },
        character: { slot: '.igs-character-layer', visible: Boolean(scene.character), character: scene.character },
        avatar: { slot: '.igs-avatar-layer', visible: readerState.avatarVisible, speaker: scene.speaker },
        dialogue: { slot: '.igs-dialogue-layer', visible: true, speaker: scene.speaker, text: scene.text },
        hud: { slot: '.igs-hud-layer', toolbar: { slot: '.igs-toolbar', placement: readerState.toolbarPlacement, layout: readerState.toolbarLayout } },
        choice: { slot: '.igs-choice-layer', visible: false },
        system: { slot: '.igs-system-layer', visible: true },
    },
}
```

3. 连接 stage renderer
   - `createStageRenderer(layerController)` 保持入口不变。
   - `render(scene, options)` 内部调用 `createReaderState()` 与 `createStageModel()`。
   - 调用 `layerController.render(stageModel)`。
   - 返回值改为 `{ ok:true, stage: stageModel, layers: layerResult }` 或兼容当前 `{ ok:true }` 并附加 `stage`，避免破坏已有测试。

4. 调整 layer controller
   - 让 `createLayerController(layers)` 接收 stage model。
   - 对每个 layer 调用 `layer.render(stageModel)`；如旧 layer 只读 `scene`，可在第一阶段向 layer 传 `{ scene: stageModel.scene, stage: stageModel }` 或兼容直接传 `stageModel.scene`，具体以现有测试最小改动为准。
   - 返回结构增加 `renderedLayers`，记录实际调用过的 layer key，便于模拟测试断言。

5. 增加 visual fixtures
   - `app/fixtures/visual/reader-settings.json`：
     - 包含 `pc`、`mobile`、`web`、`fullscreen` 四类 reader settings。
     - mobile 默认 `toolbarLayout: "vertical"`。
     - pc 默认 `toolbarLayout: "horizontal"`。
   - `app/fixtures/visual/stage-model.json`：
     - 包含一条普通背景+立绘场景预期。
     - 包含一条生图优先场景预期。
     - 不放真实图片，只用 `placeholder://` 或 `prompt://`。

6. 增加 gate 测试
   - `gate:visual-reader-state:normalizes-settings`
     - 断言 pc/mobile/web/fullscreen 的 toolbar layout、placement、dialogueStyle 和 CSS 变量被归一化。
   - `gate:visual-responsive-layout:desktop-portrait-landscape`
     - 断言 `getResponsiveLayout({ width: 1280, height: 720 }, { mode:'pc' }) === 'desktop'`。
     - 断言 `getResponsiveLayout({ width: 390, height: 844 }, { mode:'web', isMobile:true }) === 'mobile-portrait'`。
     - 断言 `getResponsiveLayout({ width: 844, height: 390 }, { mode:'fullscreen', isMobile:true }) === 'mobile-landscape'`。
   - `gate:visual-slots-contract:stage-model`
     - 用 `checkStyleContract(stageModel)` 断言必需 slot 与 reader bridge attribute 不缺失。
   - `gate:simulation:reader-stage-renders-dialogue-and-toolbar`
     - fake host message 刷新后，断言 `result.render.stage.layers.dialogue.text` 等于解析后的正文。
     - 断言 `.igs-toolbar` 存在，`data-igs-toolbar-layout` 与 `data-igs-toolbar-placement` 存在。
   - `gate:simulation:reader-stage-generated-image-slot`
     - 含生图段消息刷新后，断言 `generated.visible === true`，`visualMode === 'generated-first'`。

7. 更新文档与版本记录
   - 实施代码时把 `app/package.json` 版本升到 `0.2.3`。

## 数据流

```text
fake TavernHelper message / API message
  -> parseSceneText()
  -> resolveScene()
  -> resolveVisualMode()
  -> createReaderState()
  -> createStageModel()
  -> layerController.render(stageModel)
  -> state.lastRender.stage
  -> checkStyleContract(stageModel)
  -> npm run gate
```

## 验收

实施完成后运行：

```powershell
cd "<当前项目目录>\app"
npm run gate
```

本轮不要求运行：

```powershell
npm run package:loader
```

除非实施时同步修改了 `app/dist`、`loader/igs-loader.js` 或 `loader/igs-loader.json`。

## 通过标准

- `npm run gate` 全部通过。
- `createReaderState()` 能归一化 pc/mobile/web/fullscreen 的 reader settings。
- `createStageModel()` 输出包含 `style-contract.js` 已要求的核心槽位：`.igs-stage`、`.igs-background-layer`、`.igs-generated-layer`、`.igs-character-layer`、`.igs-dialogue-layer`、`.igs-hud-layer`、`.igs-toolbar`、`.igs-choice-layer`、`.igs-system-layer`。
- `createStageModel()` 的 `layers` 中预留 `.igs-effect-layer` 与 `.igs-avatar-layer`，但不要求当前 `checkStyleContract()` 必须检查它们，除非实施时同步扩展样式契约。
- `checkStyleContract(stageModel)` 返回 `{ ok: true }`。
- `refresh()` 的返回值或 `IGS.getState().lastRender` 可读到 `stage`，其中 dialogue layer 包含当前 `speaker` 与 `text`。
- 含生图段正文进入 generated layer；普通正文保持 background/character/dialogue layer。
- mobile + fullscreen/web + landscape fixtures 返回 `mobile-landscape`，竖屏 fixtures 返回 `mobile-portrait`，pc 返回 `desktop`。
- 本轮不新增 `project.json/latest/archive/tavern helper` 发布壳。
- 本轮不修改 `projects/<当前本地项目文件夹>` 本体。
- 本轮不挂载 `window.ImmersiveGalgameSystemBridge`。

## 不做范围

- 不迁移完整 Immersive Galgame System 阅读器 DOM。
- 不实现真实设置面板。
- 不实现完整工具栏按钮动作、收纳动画、toast UI 或确认弹窗。
- 不接真实 NAI / st-chatu8 / chami provider。
- 不扫描真实 SillyTavern DOM。
- 不做真实酒馆实机验真。
- 不打包发布 loader，除非后续实现中实际修改了发布链路。

## 残余风险

- 本轮 stage model 仍是可测试模型，不是真实 DOM；后续接 DOM renderer 时需要用 computed style 或截图测量替换 `proposed default`。
- `layerController` 兼容旧 layer 的传参方式需要实现时谨慎处理，避免破坏已有最小 loop 测试。
- 工具栏动作、toast、选择层和设置面板仍只是 slot/model 边界，用户可见交互体验需后续版本继续实现。
- 旧 Immersive Galgame System 的完整阅读器行为仍未逐函数迁移；需要时继续从 `projects/<当前本地项目文件夹>.zip` 只读抽取参考。
