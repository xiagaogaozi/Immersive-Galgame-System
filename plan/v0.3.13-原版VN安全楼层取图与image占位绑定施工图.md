# v0.3.13 原版 VN 安全楼层取图与 image 占位绑定施工图

## Summary

- 本施工图目标是把原版 Visual Novel 的安全取图思路借鉴进 VN：先锁定当前聊天楼层，再把 `<image>` / `image###...###` 占位放回当前楼层，最后只在当前楼层范围内等待和收集生图插件产物。
- 本轮不是重做 `v0.3.6` 的 `<image>` 图位绑定。当前 VN 已经有 `imageSlots`、`segmentImageSlots`、`reader-image-service` 槽位优先和阅读器翻页同步；本轮只补“扫描边界”和“占位触发”两块安全缺口。
- 用户可见目标：VN 不再把角色卡头像、右侧人物图、其它楼层图片或页面装饰图当作 VN 背景图；正文翻到第几段，只显示该段绑定图位自己的图，缺图时显示等待或空图状态，不偷拿无关图补位。
- 目标模式：奶龙工具箱 B. JS-Slash-Runner 纯 JS 脚本边界内的独立 VN app 工程；入口对象仍是 `window.TavernHelper`，最终导入件仍是 `loader/vn-loader.json`。
- 项目边界：继续使用 `app/` + `loader/` 独立结构；不新增 `project.json`、`latest/`、`archive/`、`tavern helper/`；不运行奶龙工具箱 `pack-project`、`verify-project`、`validate`、`check-refs`。
- 风险级别：本策划书为 `R1` 文档计划；实施轮会修改 `generated-images`、`host`、`scene`、`api` 和模拟测试，按 `R3` 执行；若同步版本、dist、loader、提交、推送和标签，提升为 `R4`。

### 未确认项

- 尚未确认 st-chatu8 或 chami 在所有版本中都能因为隐藏 `.vn-img-ph` 立即生成图片；因此实现轮必须把占位注入做成可重复、可清理、可禁用的增强能力，而不是唯一取图来源。
- 尚未确认所有用户环境的 SillyTavern 楼层 DOM 都含 `.mes_text`；实现轮必须保留“找不到 `.mes_text` 时只跳过占位注入”的温和失败，不得因此扫全页面。
- 尚未确认真实酒馆中角色卡图的 class 名一定包含 `avatar`；因此不能继续依赖 `isLikelyHostDecorImage()` 作为主要安全线，只能作为最后过滤。

## 当前问题结论

当前 VN 的图位绑定主体已经存在，问题不在“不会读 `<image>`”。证据 E16、E18、E19 显示：VN 已经会解析图位、传递图位，并让阅读器按段落映射图位。

真正风险在图片来源边界。证据 E20 显示：`resolveContextRoots()` 在找不到当前消息根时会退回全页面。证据 E21 显示：generic selector 包含 `img[src]` 和 `background-image`。这两点叠加时，VN 会像拿大网兜鱼一样把页面里不属于当前楼层的图捞进来，角色卡头像、侧栏图或其它楼层图就有机会混进 `imageState`。

原版 VN 的可借鉴点不是它的补位规则，而是它的安全顺序：先找到当前 `.mes`，把 `<image>` 占位藏回当前 `.mes_text`，再只在当前 iframe document 和当前 `.mes` 内找 `st-chatu8/chatu8` 图片。VN 应借鉴这个边界，但不照搬原版“当前图缺了就向前/向后找图”的补位，因为 VN 当前目标是严格图位绑定，缺图宁可等待，不拿无关图凑数。

## 施工原则

1. 先锁楼层，再找图片。
2. 有 `<image>` 图位时，找不到当前楼层就不扫图。
3. 插件图优先：`st-chatu8` / `chami` 高于普通 DOM 图。
4. 普通 DOM 图只能在当前楼层内使用。
5. 当前图位没有 URL 时保留空槽或等待状态，不偷其它图。
6. 隐藏占位只用于触发生图插件，不进入阅读正文。
7. 角色卡图、头像、侧栏图、其它楼层图永远不能进入 `imageSlots`。

## 施工步骤

### 1. 增加安全取图策略

修改 `app/src/generated-images/reader-image-service.js`：

- 新增 `resolveScopedImageRoots(message, globalObject, options)` 或扩展 `resolveContextRoots()`。
- 当 `context.imageSlots.length > 0` 或 `context.requiresMessageScope === true` 时：
  - 如果 `getMessageScopedRoots(message)` 返回非空，只返回这些 roots。
  - 如果 roots 为空，返回空数组，并在 `imageState` 中写入可诊断 reason，例如 `message-scope-not-found`。
  - 禁止退回 `getGlobalDetectionRoots(globalObject)`。
- 只有没有图位、用户关闭图片保留标签、或执行纯外部适配检测时，才允许全页面检测。
- `testImageApi()` 的外部扩展检测可以保留全页面检测，但 `collect()` 和 `regenerate()` 不能用它填图。

### 2. 给 providerContext 增加范围说明

修改 `buildProviderContext()` 和 `collectProviderImages()` 调用链：

- 增加字段：

```js
{
    roots,
    scopePolicy: {
        kind: 'message',
        hasMessageScope: roots.length > 0,
        allowGlobalFallback: false,
        hasImageSlots: imageSlots.length > 0
    }
}
```

- provider 只读取 `context.roots`，不自行改用 `document` 或 `top.document`。
- `scopePolicy.hasMessageScope === false && hasImageSlots === true` 时，provider 直接返回空数组。
- 保持 st-chatu8/chami 是可拆卸 provider，不把特定插件硬编码进核心之外的业务判断。

### 3. 拆分 DOM 图片候选等级

修改 `app/src/generated-images/dom-image-candidates.js`：

- 保留当前 chatu8 selector：
  - `img.st-chatu8-image-tag-image`
  - `[class*="st-chatu8"] img`
  - `[class*="chatu8"] img`
- 保留当前 chami selector：
  - `.tsp-generated-image`
  - `.tsp-inline-image`
  - `.tsp-image-slot img`
  - `img[src*="tsp-images"]`
  - `data-image-id` / `data-location-hash` 元数据选择器
- 收紧 generic selector：
  - 在 message-scoped roots 内允许 `.mes_text img[src]`、`.mes_text img[data-src]`、`img[src^="blob:"]`、`img[src^="data:image"]`。
  - 在全局 roots 内禁止 `img[src]`、`img[data-src]`、`[style*="background-image"]` 作为图位候选。
  - `background-image` 仅保留给明确插件容器或带 `data-location-hash/data-image-id` 的节点。
- `isLikelyHostDecorImage()` 只作为兜底过滤，不作为主防线。

### 4. 借鉴原版 VN 的隐藏占位注入

在 `app/src/host/tavern-helper-adapter.js` 新增宿主能力，建议命名：

```js
export function ensureMessageImagePlaceholders(message, imageSlots = []) {}
export function removeMessageImagePlaceholders(message) {}
```

实现要求：

- 只操作当前 message 的 `.mes_text`。
- 如果没有 `.mes_text`，返回 `{ ok: false, reason: 'message-text-root-not-found' }`，不抛错。
- 生成一个隐藏容器：
  - `class="vn-img-ph vn-image-placeholder"`
  - `data-vn-placeholder="1"`
  - `data-vn-image-placeholder="1"`
  - `style` 使用原版同类零尺寸隐藏策略：`position:absolute;width:0;height:0;overflow:hidden;opacity:0;pointer-events:none;font-size:0`
- 容器文本来自 `imageSlots[].rawBlock`，用换行拼接。
- 同一楼层重复调用时先复用或替换旧容器，避免重复触发生图。
- 注入后触发轻量 DOM 变更事件，兼容依赖 DOM 变动扫描的生图插件。
- `message-source.js` 已在 `getVisibleMessageTextFromElement()` 清理 `.vn-img-ph`，实施轮需保留并增加 `.vn-image-placeholder` 清理。

### 5. 接入占位注入时机

修改 `app/src/api/visual-novel-compat.js` 或 `reader-image-service.collect()` 的上层调用：

- 构建 payload 并拿到 `imageSlots` 后，如果 `imageSlots.length > 0` 且 message context 存在，先调用 `hostAdapter.ensureImagePlaceholders(messageId, imageSlots)` 或等价方法。
- 占位注入失败不能阻断阅读器打开，只记录 warning。
- 占位注入后再调用 `collectMessageImages()`，让 st-chatu8/chami 有机会在当前楼层内生成或暴露图片。
- 如果注入成功但暂时没有图片，`imageState.slots` 仍保持完整图位，`currentUrl` 为空，阅读器显示等待状态。

### 6. 收紧图位填充规则

修改 `buildSlottedImageState()`：

- 继续允许：
  - `slotIndex` 精确匹配。
  - `locationHash` 精确匹配。
  - `imageId` 精确匹配。
  - 候选数量等于图位数量时按顺序填槽。
- 禁止：
  - 来自全局 roots 的 generic 图片填槽。
  - 只有 1 张无元数据 generic 图时自动填入当前图位。
  - 当前图位无图时用 `unboundImages` 或其它 slot 图当 `displayUrl`。
- `displayUrl` 应与当前 slot 自己的 `url` 一致；若当前 slot 无 URL，`displayUrl` 为空。
- 仅在无图位旧模式下保留旧 `images[currentIndex]` 行为。

### 7. 重绘按钮也锁定当前楼层

修改 `resolveRegenerateButton()` / `findMessageRegenerateButton()` 相关链路：

- 重绘按钮候选只来自 `getMessageScopedRoots(message)`。
- 优先按当前 slot 的 `locationHash`、`imageId`、`buttonIndex`、`slotIndex` 匹配。
- 找不到当前楼层按钮时返回 `regen-button-not-found`，不扫全页面按钮。
- 如果按钮数量与图位数量相等，可按图位顺序兜底。

### 8. 日志与诊断

新增可测试诊断字段，不要求新增 UI 控件：

- `imageState.scopeKind`: `message` / `none` / `legacy-global`
- `imageState.scopeOk`: `true` / `false`
- `imageState.reason`: `message-scope-not-found`、`placeholder-injected`、`placeholder-skipped`、`no-provider-images-yet`
- `imageState.diagnostics.providerCounts`: 记录 chatu8/chami/generic 各自候选数量。

这些字段只进 snapshot / 测试 / console 诊断，不改变用户设置页布局。

## 通过标准

- 有 `<image>` 图位时，VN 只能从当前消息楼层及其 iframe document 收图。
- 当前消息根找不到时，VN 不扫全页面，不显示角色卡图、头像图、侧栏图或其它楼层图。
- 隐藏占位只存在于当前 `.mes_text`，不会出现在阅读器正文、可见文本、发送上下文或历史摘要里。
- st-chatu8/chami 插件图在当前消息内仍能被优先识别。
- generic 普通图只在当前楼层内作为最后兜底，且不能越过图位绑定规则。
- 当前图位缺图时，阅读器保持该图位的进度和动作上下文，不显示其它图位或无关图。
- 重绘、保存、轮询都只针对当前图位和当前楼层。
- `npm run test`、`npm run simulate`、`npm run gate`、`npm run build:loader` 全部通过。
