# characters 模块契约

## 职责

- 管理角色立绘、头像和说话人匹配。
- 规则为：说话人名字 + 情绪 -> 立绘 / 头像。
- 支持生图层中以头像方式显示当前说话人。
- 立绘与头像资源句柄由 `media` 提供。

## 数据类型

- `character-pack`
- sprite resource
- avatar resource

## 匹配顺序

1. 精确说话人 + 精确情绪。
2. 精确说话人 + 默认情绪。
3. 别名匹配 + 情绪。
4. 全局默认头像或立绘。

## 禁止

- 禁止直接管理 Blob 生命周期，资源缓存交给 `media`。
- 禁止把说话人解析规则写死在视觉层。
